DATABASE PRODUCT ID MISMATCH ISSUE (Checkout Security Flow)
============================================================

STATUS / ISSUE:
During checkout, the browser console/network logs report "Product ID 1 not found in catalog".

WHY IT OCCURS:
1. The client storefront UI (apps/client/src/components/ProductList.tsx) currently uses a list of hardcoded, static mock products with IDs 1 to 10.
2. The checkout flow implements a security check on the backend (apps/payment-service/src/controllers/payment.controller.ts) which fetches products from the PostgreSQL database using Prisma (prisma.product.findMany) to verify their price and prevent client-side price tampering.
3. Because the local PostgreSQL database is empty or does not have products matching IDs 1 to 10, the query returns an empty result, triggering the security check block.

PROPOSED RESOLUTION FOR FUTURE:
1. Populate/Seed the PostgreSQL database with the matching product details and specify their IDs as 1 through 10.
2. A seeder script "seed.js" can be executed using Prisma in the payment-service to create these mock products, categories, and image mappings.
3. Alternatively, connect the frontend storefront UI (ProductList.tsx) to query products from the database API rather than using static/temporary data.
